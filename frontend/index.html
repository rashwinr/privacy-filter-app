Quick start (curl)


TOKEN=$(curl -s -X POST /api/demo-token \
  -H "Content-Type: application/json" \
  -d '{"name":"Your Name","email":"you@example.com"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

curl -X POST /api/redact \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/document.pdf"


          
Quick start (Python)


import requests

# 1. Get a demo token
resp = requests.post(
    "https://privacy-filter.tanuh.org/api/demo-token",
    json={"name": "Your Name", "email": "you@example.com"},
)
token = resp.json()["access_token"]
headers = {"Authorization": f"Bearer {token}"}

# 2. Redact a document
with open("document.pdf", "rb") as f:
    r = requests.post(
        "https://privacy-filter.tanuh.org/api/redact",
        headers=headers,
        files={"file": ("document.pdf", f, "application/pdf")},
        timeout=600,
    )
    r.raise_for_status()
    result = r.json()

print(result["entity_counts"])
print("Download:", result["redacted_url"])

Limits and behaviour

          
    
            
  • Max request timeout: 600 seconds.
    • 
            
  • Concurrency per instance: 1 (large documents may queue).
    • 
            
  • Scanned PDFs are OCR'd at 200 DPI before detection.
    • 
            
  • Three detection layers run on every request: openai/privacy-filter (personal PII), dslim/bert-base-NER (organisations / locations / people), and deterministic regex rules (tax IDs, IFSC, IBAN, SSN, EIN, Aadhaar, emails, URLs, honorific-prefixed names, labeled fields).
    • 
            
  • Detections are best-effort. Always review redacted output before sharing.
    • 
          


          
Tuning via headers / env

          

            For self-hosted deployments, behaviour can be tuned via env vars:
            EXTRA_REDACTION_KEYWORDS,
            DISABLED_RULE_DETECTORS,
            ENABLE_NER_MODEL,
            NER_MODEL,
            PRIVACY_FILTER_OCR_DPI.
